一、Hexo的常用指令合集

Hexo：一个Node.js的静态网页生成器，常将它与Github Page搭配使用，创建个人博客网站。
hexo有许多主题，最火爆的是NexT、Matery等，复制关键词到GitHub或者baidu都能搜到。权衡利弊最终我还是选择了大多数人使用的next。

1.hexo g 或 hexo generate

该命令执行后在hexo站点根目录下生成public文件夹

2.hexo clean

把1中的public文件夹删除

3.hexo s 或 hexo server

启动服务，默认地址为http://localhost:4000/，4000端口

4.hexo d 或 hexo deploy

部署站点，在本地生成.deploy_git文件夹，并将编译后的文件上传至 GitHub

5.hexo new “”

例如： hexo new photo “my-first-blog”

上述指令执行时，Hexo 会尝试在 scaffolds 中寻找photo.md布局，若找到，则根据该布局新建文章；若未找到或指令中未指定该参数，则使用post.md新建文章。新建文章的名称在_config.yml中配置

6.删除文章

只需在本地把source/_posts文件夹下的文章源文件删除后，执行以下命令重新部署即可

hexo clean && hexo g && hexo s // 一定要先 clean，防止灵异事件发生

7.备份/迁移你的Hexo博客

备份根目录，重新安装hexo，使用备份的目录覆盖新装的hexo目录，
最后使用下面的命令直接运行即可

hexo clean && hexo g && hexo s

将公钥复制在GitHub中即可使用hexo d命令

二、Linux安装Hexo

1.安装nodejs和npm

yum install epel-release -y
yum install nodejs npm -y

2.安装git

yum install git -y

3.安装hexo

npm install hexo-cli
npm install hexo

4.检查是否安装成功

[root@node1 ~]# npm list
root@ /root
├── hexo-cli@4.3.0
└── hexo@6.3.0

通过npm list可以看到hexo和hexo-cli的相关信息，说明安装成功

5.初始化网站

hexo init myhexo

网站名称myhexo，可自行定义，初始化成功后，会在当前目录生成myhexo的网站目录

cd myhexo/
npm install

6.启动hexo

#hexo默认使用4000端口，启动前需通过防火墙放开4000端口
firewall-cmd --zone=public --add-port=4000/tcp --permanent
firewall-cmd --zone=public --add-port=4000/udp --permanent
firewall-cmd --reload

hexo server

注意：hexo server命令必须在myhexo目录下才能执行。

7.访问hexo

浏览器访问：http://192.168.1.100:4000

注意：192.168.1.100是部署hexo的节点ip地址。

五、Markdown中输入空格的几种方式

1.nbsp

&nbsp; 为“不换行空格”，全称“No-Break Space”，它是最常见和我们使用最多的空格，大多数的人可能只接触了 ，它是按下space键产生的空格。在HTML中，如果你用空格键产生此空格，空格是不会累加的（只算1个）。要使用html实体表示才可累加，该空格占据宽度受字体影响明显而强烈。

2.ensp

&ensp; 为“半角空格”，全称是“En Space”，en是字体排印学的计量单位，为em宽度的一半。根据定义，它等同于字体度的一半（如16px字体中就是8px）。名义上是小写字母“n”的宽度。此空格传承空格家族一贯的特性：透明的。此空格有个相当稳健的特性，就是其占据的宽度正好是1/2个中文宽度，而且基本上不受字体影响。

3.emsp    

&emsp; 为“全角空格”，全称是“Em Space”，em是字体排印学的计量单位，相当于当前指定的点数。例如，1em在16px的字体中就是16px。此空格也传承空格家族一贯的特性：透明的。此空格也有个相当稳健的特性，就是其占据的宽度正好是1个中文宽度，而且基本上不受字体影响。

4.thinsp

&thinsp; 为“窄空格”，全称是“Thin Space”。我们不妨称之为“瘦弱空格”，就是该空格长得比较瘦弱，身体单薄，占据的宽度比较小。它是em的1/6宽。

5.zwnj

&zwnj; 为零宽不连字，全称是Zero Width Non Joiner，简称“ZWNJ”，是一个不打印字符，放在电子文本的两个字符之间，抑制本来会发生的连字，而是以这两个字符原本的字形来绘制。Unicode中的零宽不连字字符映射为“”（zero width non-joiner，U+200C），HTML字符值引用为：&zwnj; 。

6.zwj

&zwj; 为“零宽连字”，全称是“Zero Width Joiner”，简称“ZWJ”，是一个不打印字符，放在某些需要复杂排版语言（如阿拉伯语、印地语）的两个字符之间，使得这两个本不会发生连字的字符产生了连字效果。零宽连字符的Unicode码位是（U+200D），HTML字符值引用为：&zwj; 。

7.其他

此外，浏览器还会把以下字符当作空白进行解析：空格（&#x0020;）、Tab制表位（&#x0009;）、换行（&#x000A;）、回车（&#x000D;）、全角空格（&#12288;）等等。

发表于 2023-02-28 更新于 2023-03-01

1.安装node.js

网址：https://nodejs.org/en/download/
查看版本：node -v

2.安装Hexo

安装命令：npm install hexo
安装命令：npm install hexo-cli
查看版本：hexo -v
查看安装情况：npm list

3.新建文件夹，初始化

cd /blog
hexo init 或者 hexo init blog
npm install
hexo g（生成静态网页）
hexo s（打开本地服务器查看网页http://localhost:4000）

4.上传GitHub

新建仓库，取名"自己账号名字.github.io"，选择public公开库，最后Add a README file勾上，点击创建完成。
进入仓库点击Settings，找到GitHub Pages，点击Check it out here!将Source里的Branch分支改成main，点击Save，如果已经是这样那就不用改了。
然后就可以通过网址https://***.github.io访问自己的blog网页了。

5.配置账号、密码、SSH

打开Git Bash
git config --global user.name "你GitHub账号的名字"
git config --global user.email "你GitHub账号的邮箱"
ssh-keygen -t rsa -C "你GitHub账号的邮箱"（生成SSH Key，3次回车）
cat ~/.ssh/id_rsa.pub输出SSH Key，然后全部复制下来
然后打开GitHub点击自己的头像，再点击Settings
再点击SSH and GPG keys
然后再点击New SSH key
然后输入SSH key的名字，随便取，再输入刚才复制的SSH key，点击Add SSH key就添加好了

6.更改博客blog配置文件

打开博客blog文件夹里面的**_config.yml**文件，这是你的博客的配置文件，你可以在这更改关于博客的信息

deploy:
    type: git
    repository: https://github.com/HuiDouDaXian/HuiDouDaXian.github.io（储存库的链接）
    branch: master

保存配置文件
npm install hexo-deployer-git --save安装GitHub推送插件
安装好了以后再输入hexo g生成静态网页，再输入hexo d上传到GitHub就可以了
然后上GitHub把分支调成master就可以看到推送的博客文件了
接着把设置中的Github Page里的分支改成master就好了，如果已经是这样了那就不用改了
然后上你的博客看看吧，地址就是你的储存库名字
https://HuiDouDaXian.github.io

7.配置博客

修改博客信息
首先打开_config.yml配置文件，找到Site注释的地方，这里是博客的基本信息

更改主题
cd hexo-site
npm install hexo-theme-next安装NexT主题
npm安装的NexT主题的目录在node_modules文件夹下的hexo-theme-next文件夹中，里面包含了NexT的配置文件，将hexo-theme-next文件夹复制到themes文件夹里面，打开_config.yml配置文件，修改theme:hexo-theme-next

8.解决乱码问题

原因：由于站点配置文件没有使用utf-8编码而导致的中文乱码
将title改为中文字符，language: zh-CN。
将站点配置文件_config.yml保存成utf-8格式
重启博客 hexo clean && hexo g && hexo s
markdown文件用utf-8编码保存
使用记事本打开博客文件，不需要修改，保存的时候以utf-8的格式保存。注意保存的时候不要修改后缀，默认会改成txt文件，注意保存

9.卸载博客

npm uninstall hexo-cli -g
再找到文件夹中的blog，手动删除即可。
